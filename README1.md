# Практическое задание. Вариант 4. 0-1 Рюкзак с жадным алгоритмом (Виговская, УИБО-10-24)
Реализовать жадный приближенный алгоритм для 0-1 рюкзака. Отличие от дробного
в том, что предметы нельзя дробить.
## Как работает алгоритм?

Алгоритм ```greedy_knapsack_2_approx``` использует две стратегии выбора предметов для гарантированного приближения к оптимальному решению.

Первая стратегия - жадное решение по удельной стоимости: предметы сортируются по убыванию отношения ```стоимость/вес```, затем последовательно добавляются в рюкзак пока есть свободная вместимость. Функция ```sorted(items, key=lambda x: x[1] / x[0], reverse=True)``` выполняет эту сортировку, а цикл ```for weight, value, idx in sorted_items:``` осуществляет жадное заполнение.

Вторая стратегия - поиск самого ценного предмета: алгоритм находит один предмет с максимальной стоимостью, который помещается в рюкзак. Цикл ```for weight, value, idx in items:``` перебирает все предметы, проверяя условие ```weight <= capacity and value > best_single_value.```

Финальный выбор - сравнение двух стратегий: оператор ```if greedy_value >= best_single_value:``` выбирает лучшее из двух полученных решений, обеспечивая коэффициент аппроксимации 2.0.

## Временная сложность
Общая временная сложность: ```O(n log n)```
## Почему именно такая временная сложность?
Сортировка является наиболее вычислительно затратной операцией в алгоритме. Даже при том, что остальные этапы выполняются за линейное время ```O(n)```, общая сложность составляет ```O(n log n)``` согласно правилу доминирования в анализе алгоритмов.
